# 店舗作業タイマー

冷凍食品の店舗配送・陳列ワークの作業時間を、ストップウォッチ式で計測する個人用 PWA。
「どこで止まっているか」を per-unit（○○あたりの速さ）で発見するのが目的。

- 公開URL: https://sebanne1225.github.io/store-timer/
- データは端末の `localStorage` に保存（サーバー送信なし）
- JSON でエクスポート / インポート（バックアップ・復元）

## v0.2: 一本道フロー

1日の運用を①〜⑱のフェーズでシームレスに進める。リロードしても進行中フェーズに復帰。

1. **① タイトル** — 設定（コース変更）・一覧・入出力
2. **②③ 数値入力** — 店舗別数量 + 当日EOS。画面内テンキー常設。C店は 冷食数=合計ケース−アイス を自動計算。オリコン数は現場（⑧）で後入力
3. **④⑤ 計画** — 前日の引継ぎメモを店舗タブで確認しつつ、今日のTodo（非計測）を各店+全体で作成
4. **⑰ 積み込み（朝積みコースはここ）**
5. **⑥ 回る順番** — 当日限りの並べ替え・行かない店のスキップ
6. **⑦ 出発前チェック** — 固定3項目（全店書類 / ハンディ / 欠品報告）
7. **⑧ 店舗計測** — ステップをタップで任意順に切替スタート/再開。手修正は数字だけ（1000=10分00秒）。その他ステップ追加可（合計外）。メモ類はドロワー: ⑨学び / ⑩疑問 / ⑪店引継ぎ / ⑫全体引継ぎ（→帰社Todoコピー可）/ ⑬店Todo
8. **⑮ 次店舗へ** → 全店終わったら **⑯ 帰社時Todo**
9. **⑰ 積み込み（夜積みコースはここ）** — 翌日EOSを入力すると翌日の②へ自動転記
10. **⑱ 引継ぎ整理** — 今日の引継ぎを編集/削除して翌日④に出す内容を確定 → 1日完了

**⑭ 一覧モード**はヘッダからいつでも。過去日の閲覧・任意店の直接計測（一本道に従わない日の逃げ道）もここから。

## per-unit の式
- ケースあたり = (アイス+冷食 時間) / (アイス数+冷食数)
- アイスあたり = アイス時間 / アイス数 ／ 冷食あたり = 冷食時間 / 冷食数
- バラあたり = バラ時間 / バラ数 ／ オリコンあたり = バラ時間 / オリコン数
- S店 = 合計時間 / ケース数
- 積み込み: EOSあたり = 積み込み時間 / EOS（夜積みは翌日EOS基準）

## 店プリセット・コース
- A: 冷食 / アイス ／ C: 積下+検品 / アイス / バラ / 冷食 / 前出し / チェック+片付け / 週次(合計外) ／ S: 積下 / 検品+陳列
- コース変更（①から）で店の並べ替え・追加・削除、夜積み/朝積みの切替が可能
- 既定コース: A-K → C-T → S-T → S-F → C-A → C-N（夜積み）

計測対象は「店到着〜出発の作業時間」（車移動は測らない）。

## データ
- localStorage `storeTimer`（schema v2）。v0.1 データは自動移行（数値温存・旧メモ欄は廃止）
- 計測はエポック基準（`startAt`+累積秒）なのでスリープ・リロードで狂わない
