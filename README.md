# 店舗作業タイマー

冷凍食品の店舗配送・陳列ワークの作業時間を、ストップウォッチ式で計測する個人用 PWA。
「どこで止まっているか」を per-unit（○○あたりの速さ）で発見するのが目的。

- 公開URL: https://sebanne1225.github.io/store-timer/
- データは端末の `localStorage` に保存（サーバー送信なし）
- JSON でエクスポート / インポート（バックアップ・復元）

## 一本道フロー（v0.2〜）

1日の運用を①〜⑱のフェーズでシームレスに進める。リロードしても進行中フェーズに復帰。

1. **① タイトル** — 設定（コース変更）・一覧・入出力
2. **②③ 数値入力** — 店舗別数量 + 当日EOS。画面内テンキー常設。C店は 冷食数=合計ケース−アイス を自動計算。オリコン数は現場（⑧）で後入力。A店は冷食→アイスの順（ステップ順と一致）。項目一覧カードは廃止し「＋追加 + 📓ノート入口（未完バッジ付き）」のみ
3. **④⑤ 計画** — 📬未完の申送（消すまで毎日届く）・✓済・報告（前回分）・✅今日の項目を1画面に整理。📋ノートからコピーで過去の項目を再利用
4. **⑰ 積み込み（朝積みコースはここ）**
5. **⑥ 回る順番** — 当日限りの並べ替え・行かない店のスキップ
6. **⑦ 出発前チェック** — 固定3項目（全店書類 / ハンディ / 欠品報告）+ 🚗出発メーター（任意入力・未記入なら確認）
7. **⑧ 店舗計測** — ステップをタップで任意順に切替スタート/再開。手修正は数字だけ（1000=10分00秒）。その他ステップ追加可（合計外）。合計外ステップ（週次チェック等）は自動遷移せずタップでのみ計測。タイマー直下に数量サマリ・合計は「除外済/全部込み」の2本立て。「チェック+片付け」突入時に未完のやることがあれば通知バナー。棚割差はカード常設、ドロワー: 📓ノート / 📐棚割差 / 📓全日
8. **⑮ 次店舗へ** → 全店終わったら **⑯ 帰社時チェック**（全体の残り項目 + 🚗帰社メーター）
9. **⑰ 積み込み（夜積みコースはここ）** — 翌日EOSを入力すると翌日の②へ自動転記。時間の手修正可。項目は「＋追加 + 📓ノート入口」のみ
10. **⑱ 棚卸し** — ⏳未完（やる/申送）・✔完了・📚今日の学び・📄ノート/メモに整理。📝申送トグルで翌朝④への申し送りを切替 → 1日完了

**⑭ 一覧モード**はヘッダからいつでも。「今日の操作」から本線フェーズへの「▶続きから」・当日のみ⑰⑱への直行・📓ノート入口。過去日の閲覧・任意店の直接計測（一本道に従わない日の逃げ道）もここから。

## v0.3: メモ統合（項目 + タグ）

書き込みの単位は「項目」1 種類。タグ（✅やる / 📝申送 / 📚学び / 💭メモ / 📱LINE）を複数付けられ、書く画面ごとに既定タグが自動で入る（⑧の店画面なら その店+✅やる、⑱なら 📝申送）。

- **📝申送 = 翌朝に届く**: 未完了はそのまま翌日の項目に、完了分は「✓済・報告」として表示（Todo と引継ぎの二重概念を廃止）
- **ノート** = LINE 転記などの長文（チェックしない塊テキスト）
- **📐棚割差** = 店に紐づく永続メモ（日次リセットなし）
- **📖記録**（ヘッダ常設）= 全項目・ノート・学び・棚割差を日付横断で遡る履歴ビュー。タグチップで絞り込み
- 更新配信の可視化: バージョン表示 + 新版検知バナー（開きっぱなし PWA の旧版滞留対策）

## v0.4: 統合ノート（📓）+ 申送永続 + タイプタグ + メーター記録

「項目」「ノート」の器を `entries[]` 単一ストリームへ統合。やること・申送・学び・メモ・LINE を全部同じ器で扱う。

- **📓 ノート**（旧📖記録を統合・改名）= 日付横断の履歴ビュー兼追加窓口。範囲チップ「今日+未完申送 / 全部の日」・タグ絞込み・🏬タイプ/店チップ。④⑱の「📋 ノートからコピー」から開くとピッカーモード（タップ即コピー）になる
- **申送の永続**: 翌朝コピー（carry-over）を廃止。未完の📝申送エントリは原本のまま、消すか完了するまで毎日④に表示され続ける
- **🏬 C/A/S タイプタグ**: 「C店全部」等、店タイプ単位で効く申送・LINE 用のタグを店タグと併用可能
- **🚗 メーター記録**: ⑦出発前・⑯帰社時に走行メーターの値と記入時刻を記録（任意入力・未記入のまま進むと確認ダイアログ）。⑭一覧の日付カード・店バッジにも反映
- **④計画・⑱棚卸し**が対称の「エントリ整理画面」に統一（📬未完の申送 / ✓済・報告 / ⏳未完 / ✔完了 / 📚学び / 📄ノート・メモ）
- **⑧計測**: 常時の項目一覧カードを廃止し、数量サマリ（タイマー直下）・合計2本立て（除外済/全部込み）・「チェック+片付け」突入時の未完通知バナーに置き換え
- **⑭一覧**: 「今日の操作」から本線フェーズへの「▶続きから」・当日のみ⑰⑱への直行ジャンプを追加

## per-unit の式
- ケースあたり = (アイス+冷食 時間) / (アイス数+冷食数)
- アイスあたり = アイス時間 / アイス数 ／ 冷食あたり = 冷食時間 / 冷食数
- バラあたり = バラ時間 / バラ数 ／ オリコンあたり = バラ時間 / オリコン数
- S店 = 合計時間 / ケース数
- 積み込み: EOSあたり = 積み込み時間 / EOS（夜積みは翌日EOS基準）

## 店プリセット・コース
- A: 冷食 / アイス ／ C: 積下+検品 / アイス / バラ / 冷食 / 前出し / チェック+片付け / 週次(合計外) ／ S: 積下 / 検品+陳列
- コース変更（①から）で店の並べ替え・追加・削除、夜積み/朝積みの切替が可能
- 既定コース: A-K → C-T → S-T → S-F → C-A → C-N（夜積み）

計測対象は「店到着〜出発の作業時間」（車移動は測らない）。

## データ
- localStorage `storeTimer`（schema v4）。v0.1〜v0.3 データは自動移行（計測値温存・旧メモ欄 learn/question/handover/todos → 項目・ノート → v0.4 で entries[] に統合。carry コピー分は破棄・原本は残る）
- 計測はエポック基準（`startAt`+累積秒）なのでスリープ・リロードで狂わない
